## Overview

When an order is placed, the Recommendations Service uses an LLM to generate
three "you might also like" items personalized to the basket. The LLM call is
captured by autotel's gen-AI instrumentation, so it shows up in the catalog
as a first-class step inside [[flow|CheckoutFlow]].

This is the kind of step that **hand-written catalogs cannot describe well**:
the prompt template, the model, the token counts, and the latency only exist
at runtime. Autotel captures them, the catalog renders them.

### How it's instrumented

```ts
import { track } from 'autotel';
import { traceConsumer } from 'autotel/messaging';

export const generateRecommendation = traceConsumer({
  system: 'kafka',
  destination: 'orders.events',
})(() => async (msg: OrderPlacedMessage) => {
  const result = await openai.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: buildPrompt(msg),
  });

  track('recommendation.generated', {
    orderId: msg.id,
    model: 'gpt-4o-mini',
    usage: {
      promptTokens: result.usage.prompt_tokens,
      completionTokens: result.usage.completion_tokens,
    },
    _autotel: { channel: 'orders.events', producer: 'RecommendationsService' },
  });
});
```
